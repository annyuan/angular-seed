Forms

Although we're not using Twitter Bootstrap's form styles, I think we're still using the gem simple_form to create our forms, which generates forms that conform to Bootstrap's markup conventions. So if you're building a form from scratch, I guess it would make sense to try to follow those conventions until we decide to switch form gems?

  • Inputs

    simple_form wraps any inputs, checkboxes, and radio buttons in a div with the class control-group.

              
%form
  .control-group.string
    %label.control-label
      %abbr *
      Name
    .controls
      %input{:type => "text"}

  • =control-group

    Use this mixin to configure the control-group div contents.

      Optional arguments:
    • $replace-labels-with-placeholder: false

      Pass in true to hide the .control-label for input fields when the browser supports placeholder text.

    • $control-group-size: false

      (quoted string) Pass in a member of the $control-group-sizes list to configure font size and spacers.

    • $block-help: false

      Pass in true to block display error help text.

    • $horizontal-labels: false

      Pass in true to display .control-label divs for input fields and input fields side by side.

    • $horizontal-label-width: 5em

      (any valid CSS width value) If $horizontal-labels is set to true, then the mixin will output a width for the .control-label divs.

form:first-of-type
  +control-group($replace-labels-with-placeholders: true)
