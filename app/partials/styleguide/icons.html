Icons

  • Symbol Set

    We're using Symbol Set's SSStandard and SSSocial.

    Icons are not included directly in the markup very often since they typically are just used for presentational enhancement. One exception is the icon button, where the icon is meaningful enough in appearance that no text needs to appear with it for the user to know what clicking on it will do.

    mail 
              
.ss-icon mail

%button.ss-icon{:href => "mailto:abc@example.com"} mail

  • =symbol

    Inserts an icon as generated content on the element.

      Required arguments:
    • $symbol

      (quoted string) The name of the icon.

      Optional arguments:
    • $placement: 'before'

      (quoted string) Pass in 'after' to include the symbol in the after pseudo element.

    This mixin may be used to enhance text, for example in buttons.

    Call home 
              
.button
  +symbol('phone')

  • =circled-symbol

    Gives the icon a colored circle background.

      Required arguments:
    • $size-in-lines

      (integer) This number will be multiplied by $base-line-height to generate the dimensions of the circle in pixels.

    • $bgc

      (color variable or code) The color of the background.

    Alert
              
#alert
  +symbol('alert', $color: white, $size: .65em)
  &:before
    +circled-symbol(6, $c-red)
