Buttons

  • button tags and the button class

    Since there is a button tag, the button class deserves to be in the markup. They come with minimal styles, but if they are being included within a frequently used component (such as a form), then they should hopefully have additional style settings pre-applied.

    And me 
              
%button Click me
%a.button And me

  • Disabled buttons

              
%button.disabled Disabled

  • =button

    This mixin can be used to configure button colors and sizes. There shouldn't be many cases where in the CSS we have to include this mixin in addition to extending the base button class, since the things that need to look like buttons are typically button tags or items that have the button class on them.

      Optional arguments:
    • $button-color: false

      (quoted string) The name of an item from the $button-colors list.

    • $button-size: false

      (quoted string) The name of an item from the $button-sizes list.

    • $c: false

      The color of the text.

    • $c-hover: false

      The color of the text on hover. Most of the time the color of the text does not change on hover.

    • $bgc: false

      The background color of the button.

    • $bgc-hover: false

      The background color of the button on hover.

    • $font-size: false

      The font-size of the text.

    • $lines: false

      (integer) The height of the button in terms of multiples of $base-line-height.

    • $lines-horizontal-padding: false

      (integer) The horizontal padding of the button in terms of multiples of $base-line-height.

              
%button:first-of-type
  +button($button-color: 'green', $button-size: 'medium')

%button:nth-of-type(2)
  +button($button-size: 'medium', $c: white, $bgc: $c-red, $bgc-hover: lighten($c-red, 10%))

%button:nth-of-type(3)
  +button($button-color: 'orange', $font-size: 11px, $lines: 7, $lines-horizontal-padding: 3)
